## Type-Enum Mapping Decision

FP's `type` field vocabulary is unconfirmed (endpoint newly probed). Mapping is conservative:
- `"material"/"materials"/"parts"/"part"` → `"material"`
- `"equipment"/"equip"` → `"equipment"`  
- Everything else (including `"service"/"services"/"labor"`, unknown strings, and null) → `"service"` (most generic value)

Unknown FP type strings are tallied and logged as a WARN at end of run so operators can see them in logs without any truncation being silent.
