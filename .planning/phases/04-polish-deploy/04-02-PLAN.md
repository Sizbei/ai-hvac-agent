Create `PRIVACY.md` at project root documenting the application's AI data handling practices. Include these sections:

**1. Overview**: This application uses OpenAI's GPT-4o API for customer service chat and structured data extraction for HVAC service requests.

**2. Data Sent to OpenAI**: 
- Chat messages (user questions about HVAC issues)
- Conversation history for context (within a single session)
- System prompts for extraction guidance
- No PII is sent in system prompts. Customer-provided PII (name, phone, email, address) appears only in user messages and extraction results.

**3. OpenAI API Data Usage Policy**:
- OpenAI API does NOT use customer data to train models (as of March 2023 policy update)
- API data is retained by OpenAI for up to 30 days for abuse monitoring, then deleted
- Reference: https://openai.com/enterprise-privacy/
- Reference: OpenAI API Data Usage Policies page

**4. Data Processing Agreement (DPA)**:
- OpenAI offers a DPA for API customers
- Organizations should execute OpenAI's DPA before processing personal data
- DPA available at: https://openai.com/policies/data-processing-agreement/

**5. Training Opt-Out**:
- API usage is already opted out of training by default
- No additional configuration needed
- This is distinct from ChatGPT consumer product (which has different defaults)

**6. Application-Level Data Protection**:
- PII is encrypted at rest using AES-256-GCM (column-level encryption)
- PII fields are redacted in application logs via Pino redaction
- Sessions expire after 24 hours of inactivity
- All data purged after 90-day retention period
- Multi-tenant isolation via organization_id scoping

**7. Recommendations for Production**:
- Execute OpenAI DPA before going live
- Review OpenAI's current data retention policies periodically
- Consider enabling OpenAI's zero-retention option if available for your plan
- Ensure ENCRYPTION_KEY is unique per environment and stored securely

**vercel.json** - Create at project root:

```json
{
  "crons": [
    {
      "path": "/api/cron/cleanup",
      "schedule": "0 3 * * *"
    }
  ]
}
```

This schedules the session cleanup cron to run daily at 3:00 AM UTC. Vercel automatically sends the `Authorization: Bearer <CRON_SECRET>` header to cron endpoints.

No other vercel.json configuration needed -- Next.js 16 on Vercel uses sensible defaults for framework detection, build command (`next build`), and output directory.

**DEPLOY.md** - Create at project root with a complete deployment runbook:

**1. Prerequisites**:
- Vercel account (https://vercel.com)
- Neon account (https://neon.tech)
- OpenAI API key (https://platform.openai.com/api-keys)
- GitHub repository connected to Vercel

**2. Neon PostgreSQL Setup**:
- Create a new Neon project (region: closest to your users)
- Create a database named `hvac_prod` (or use the default `neondb`)
- Copy the connection string from the Neon dashboard (pooled connection string recommended for serverless)
- Format: `postgresql://user:password@ep-xxx.region.neon.tech/dbname?sslmode=require`

**3. Run Migrations Against Production**:
```bash
DATABASE_URL="your-neon-connection-string" npm run db:migrate
```
This runs all migrations in `drizzle/` against the production database.

**4. Seed Production Data**:
```bash
DATABASE_URL="your-neon-connection-string" npm run db:seed
```
This creates the demo organization, admin user (admin@demo-hvac.com / admin123), and three technicians. IMPORTANT: Change the admin password after first login in production.

**5. Environment Variables** (set in Vercel Dashboard -> Settings -> Environment Variables):

| Variable | Description | Example |
|----------|-------------|---------|
| `DATABASE_URL` | Neon PostgreSQL connection string | `postgresql://...@ep-xxx.neon.tech/hvac_prod?sslmode=require` |
| `OPENAI_API_KEY` | OpenAI API key | `sk-proj-...` |
| `ENCRYPTION_KEY` | 32-byte hex key for AES-256-GCM PII encryption | Generate with: `openssl rand -hex 32` |
| `JWT_SECRET` | Secret for admin JWT tokens | Generate with: `openssl rand -hex 32` |
| `CRON_SECRET` | Secret for cron endpoint authentication | Generate with: `openssl rand -hex 32` |
| `NODE_ENV` | Set to production | `production` |
| `LOG_LEVEL` | Production log level | `info` |

**6. Deploy to Vercel**:
```bash
# Option A: Connect GitHub repo in Vercel dashboard (recommended)
# Push to main branch triggers automatic deploy

# Option B: CLI deploy
npx vercel --prod
```

**7. Post-Deploy Verification**:
- Visit `https://your-app.vercel.app` -- landing page loads
- Click "Get Help Now" -- chat session starts
- Send a message -- AI responds with streaming
- Visit `https://your-app.vercel.app/admin/login` -- login page loads
- Login with admin@demo-hvac.com / admin123
- Dashboard shows stats cards and request queue

**8. Security Checklist**:
- [ ] Change default admin password

- [ ] Verify all env vars are set in Vercel (missing vars cause runtime errors)
- [ ] Execute OpenAI DPA (see PRIVACY.md)
- [ ] Verify cron job appears in Vercel dashboard (Settings -> Cron Jobs)
