1. Create src/hooks/use-chat-session.ts -- the main orchestration hook:
   ```typescript
   'use client';
   import { useState, useEffect, useCallback, useRef } from 'react';
   import { useChat } from '@ai-sdk/react';
   import { TextStreamChatTransport } from 'ai';
   import type { SessionState } from '@/lib/ai/state-machine';
   import type { ExtractionResult } from '@/lib/ai/extraction-schema';
   import { REQUIRED_EXTRACTION_FIELDS } from '@/lib/ai/extraction-schema';
   import type { ChatMessage, ExtractionField } from '@/lib/types/chat';
   ```

   The hook manages three concerns:
   
   A) **Session creation**: On mount, call POST /api/session to create a session (which sets the httpOnly cookie). Track sessionId and loading state. If session already exists (GET /api/session returns data), restore messages from it.
   
   B) **Chat streaming**: Use the AI SDK useChat hook with TextStreamChatTransport. CRITICAL: The backend /api/chat expects `{ message: string }` in the body, NOT the default useChat format of `{ messages: UIMessage[] }`. Configure the transport with `prepareSendMessagesRequest` to transform the request:
   ```typescript
   const transport = new TextStreamChatTransport({
     api: '/api/chat',
     prepareSendMessagesRequest: ({ messages: msgs, ...rest }) => {
       // Extract the last user message content from parts
       const lastUserMsg = msgs.filter(m => m.role === 'user').at(-1);
       const content = lastUserMsg?.parts
         ?.filter((p): p is { type: 'text'; text: string } => p.type === 'text')
         .map(p => p.text)
         .join('') ?? '';
       return {
         ...rest,
         body: { message: content },
       };
     },
   });
   ```
   
   C) **Session state polling**: After each message exchange completes (useChat status goes from 'streaming' to 'ready'), fetch GET /api/session to get updated session status and extraction data (stored in session.metadata as JSON). Parse extraction data from the metadata field.
   
   The hook returns:
   ```typescript
   {
     messages: ChatMessage[];        // Converted from useChat UIMessages
     status: SessionState;           // From GET /api/session polling
     isStreaming: boolean;           // useChat status === 'streaming'
     extraction: ExtractionResult | null;  // From session metadata
     extractionFields: ExtractionField[]; // Derived from extraction
     error: string | null;
     isLoading: boolean;             // Session creation in progress
     sendMessage: (text: string) => void;  // Wraps useChat sendMessage
     escalate: () => Promise<void>;
     confirm: (data: ExtractionResult) => Promise<{ referenceNumber: string }>;
   }
   ```
   
   Key implementation details:
   - Convert UIMessage[] (from useChat with .parts) to ChatMessage[] (with .content string) using a useMemo:
     ```typescript
     const chatMessages: ChatMessage[] = useMemo(() =>
       messages.map(m => ({
         id: m.id,
         role: m.role as 'user' | 'assistant',
         content: m.parts
           .filter((p): p is { type: 'text'; text: string } => p.type === 'text')
           .map(p => p.text)
           .join(''),
       })),
       [messages]
     );
     ```
   - Extract extraction fields status from ExtractionResult:
     ```typescript
     const extractionFields: ExtractionField[] = useMemo(() => [
       { key: 'issueType', label: 'Issue Type', collected: extraction?.issueType != null },
       { key: 'urgency', label: 'Urgency', collected: extraction?.urgency != null },
       { key: 'address', label: 'Address', collected: extraction?.address != null && extraction.address.length > 0 },
     ], [extraction]);
     ```
   - The `sendMessage` wrapper calls useChat's sendMessage with the user text as a new user message
   - Use useEffect to poll session state when useChat status changes to 'ready' (after streaming finishes)
   - The `escalate` function: POST /api/session/escalate, then update local status to 'escalated'
   - The `confirm` function: POST /api/session/confirm with the extraction data, returns { referenceNumber }
   - Guard sendMessage: disable if status is terminal, submitted, or isStreaming

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | S (Spoofing) | use-chat-session.ts | mitigate | Session auth via httpOnly cookie (set by POST /api/session in Phase 1); no token in JS |
| T-02-06 | T (Tampering) | confirmation-dialog.tsx | mitigate | Server-side Zod validation on /api/session/confirm rejects invalid data; client just submits what server extracted |

| T-02-08 | D (Denial of Service) | use-chat-session.ts | mitigate | Server-side rate limiting on /api/chat and /api/session (Phase 1); client disables input during streaming |
