**src/lib/admin/queries.ts:**
Create all database query functions. Each function takes `organizationId` as first parameter and uses `withTenant`.

- `getRequests(orgId, filters)` — SELECT from serviceRequests LEFT JOIN users (on assignedTo) with optional status filter, ordered by createdAt DESC, with limit/offset pagination. Decrypt customerNameEncrypted for list display. Return `{ requests: AdminRequest[], total: number }`.

- `getRequestById(orgId, requestId)` — SELECT serviceRequest LEFT JOIN users. Decrypt ALL encrypted fields (name, phone, email, address). Also SELECT messages WHERE sessionId matches, ordered by createdAt ASC. Return `AdminRequestDetail | null`.

- `assignTechnician(orgId, requestId, technicianId)` — UPDATE serviceRequests SET assignedTo, status='assigned', updatedAt=now() WHERE id AND withTenant. Return updated request or null.

- `getTechnicians(orgId)` — SELECT from users WHERE role='technician' AND withTenant, ordered by name ASC. Return `TechnicianRecord[]`.

- `createTechnician(orgId, input: CreateTechnicianInput)` — INSERT into users with role='technician', hash password with bcrypt (salt 12). Return `TechnicianRecord`.

- `updateTechnician(orgId, techId, input: UpdateTechnicianInput)` — UPDATE users SET (name?, email?, isActive?), updatedAt=now(). Return `TechnicianRecord | null`.

- `getDashboardStats(orgId)` — 4 COUNT queries:
  - pending: WHERE status='pending'
  - assignedToday: WHERE status='assigned' AND updatedAt >= start of today (UTC)
  - inProgress: WHERE status='in_progress'
  - completedToday: WHERE status='completed' AND completedAt >= start of today (UTC)
  All scoped by withTenant. Return `DashboardStats`.

For date calculations, use `new Date()` set to UTC midnight: `new Date(new Date().toISOString().split('T')[0] + 'T00:00:00.000Z')`.

Use `eq`, `and`, `sql`, `count`, `desc` from drizzle-orm. Use `gte` for date comparisons.
